## 🎯 今天的主軸｜開放權重模型正式擠進第一梯隊——「能下載的」第一次追上了「只能租的」

**共同變化**：過去一年，大家默認真正最強的模型都是閉源的，你只能透過 API「租」它。這週被兩件事同時鬆動：7/16 中國 Moonshot 丟出 **Kimi K3**（2.8 兆參數、承諾 7/27 開放權重），在多個獨立評測排進全球前三、前四，追平 Opus 4.8；同一天 NVIDIA 與日本政府宣布的國家級 AI 基建，也刻意把基礎模型做成開放權重。

**為什麼放在一起看**：把 Kimi K3、NVIDIA×日本國家基建、以及上週 Thinking Machines 開源的 Inkling（975B）放在一起，會看到同一個方向——頂尖能力正在跟「可下載、可自架、可改」綁在一起。新創、中國大廠、國家政府三種玩家不約而同押注開放權重，這是一次集體轉向。

**過去 vs. 現在**：過去開源大概落後前沿半年到一年；現在 Kimi K3 在 Artificial Analysis 智慧指數排全球第三（Elo 約 1547），只輸 Fable 5 和 GPT-5.6 Sol，把 Opus 4.8 擠到後面，部分編碼評測甚至勝過 Opus 4.8。價格則是輸入 $3、輸出 $15，約 GPT-5.6 Sol（$5／$30）的六成。

**背後邏輯**：Kimi K3 是 MoE 架構，總共 2.8 兆參數，但每個 token 只點亮 896 個專家裡的 16 個（約 1.8%）——像有 896 位專科醫生的醫院，每次只轉診最相關的 16 位，養得起龐大知識庫、跑起來卻不用全院動員。對中國廠商，開放權重是在晶片受限下「用生態換影響力」；對 NVIDIA×日本，則是國家不想被單一美國供應商綁死。

**對你的工作**：技術主管多了一個真實選項——把敏感或高量負載跑在自架開放權重模型上，成本可控、資料不出境，能力不再明顯讓步；開發者多一個 $3／$15、Opus 級的 API，編碼場景值得實測，7/27 後還能自己微調；知識工作者短期未必直接用到，但價格戰會把你正在用的工具一起往下拉。

**限制與待驗證**：①權重還沒真的釋出（Moonshot 說 7/27），在那之前仍只能 API 用；②部分評測是廠商自跑、用不同框架，跨模型比較要打折；③2.8 兆參數自架門檻極高，「能下載」不等於「你跑得動」；④真實長時 agent 任務、工具呼叫穩定度仍需各團隊自測。

**記住這句**：前沿模型的「門票」，這週第一次不再只發給閉源巨頭——但「開放權重」目前給你的是議價能力和退路，還不是可以立刻自架上線的成品。

> 👤 人物觀點／延伸解讀：獨立開發者 Simon Willison 用「畫一隻騎腳踏車的鵜鶘 SVG」測 Kimi K3，發現它「很會想」——為畫一張圖用掉 13,241 個推理 token（單張約 25 美分），視覺理解表現也好；但他提醒這類趣味測試越來越測不出模型在長對話 agent 工具呼叫的真實實力。（人物觀點，非官方數據）

## 📖 今日概念｜開放權重（Open Weights）

**一句定義**：把訓練好的模型「參數檔」公開讓人下載、自架、微調，但不一定公開訓練資料與完整訓練程式碼。

**生活化比喻**：像餐廳把「做好的醬料」整罐送你，你能帶回家自己加菜、無限次使用——但食譜和原料來源沒給。這跟「只能到店裡點餐」（閉源 API）不同，也跟「連食譜和原料清單都公開」（真正開源）不一樣。說回技術：你拿到能跑、能改的權重，卻未必能完全「重現」它是怎麼被訓出來的。

**工作中的例子**：金融或醫療團隊把開放權重模型架在自家機房，敏感資料完全不出境，還能用內部資料微調出專屬版本——這是純 API 服務很難給的控制權。

**常見誤解**：「開放權重＝開源＝免費隨便用」。三件事要分開看：很多開放權重模型的授權有商用限制；「開放權重」不等於「開源」（後者連資料與程式碼都公開）；2.8 兆參數這種規模，自架硬體成本一點都不便宜。

**與今天主軸的關係**：主軸之所以重要，正是因為「開放權重」這一欄第一次同時擁有「前沿能力」與「可自架控制」——過去你只能二選一。

- 你手上哪些工作負載，是「資料敏感、用量又大」到值得認真評估自架開放權重的？又有哪些其實丟給便宜 API 就好、不必扛自架的維運重量？
- 當一個能力接近的模型來自你無法完全信任其供應鏈的地區，你會怎麼在「成本誘因」與「治理／合規風險」之間畫線？這條線該由工程決定，還是法務與管理層一起定？
